# M15：综合评测 & 与 MinerU Benchmark 对齐

## 目标

建立与 MinerU 可比较的综合评测体系，在 M9-M14 全部完成后，使用标准化测试集对比 knot-pdf 与 MinerU 的解析质量，找出差距并制定后续改进方向。

## 依赖

- M9 ~ M14（所有增强功能就绪）

## 交付物

- [ ] 标准化评测数据集（覆盖多种文档类型）
- [ ] 自动化评测脚本
- [ ] knot-pdf vs MinerU 的详细对比报告
- [ ] 质量差距分析与后续改进计划

---

## Checklist

### 1. 评测数据集

基于公开数据集 + 自建样本，覆盖以下场景：

| 场景                      | 数量 | 来源          |
| ------------------------- | ---- | ------------- |
| 英文学术论文（单栏）      | 10   | arXiv         |
| 英文学术论文（双栏）      | 10   | arXiv / IEEE  |
| 中文财报 / 年报           | 5    | 上市公司公告  |
| 合同/法律文档             | 5    | 自建          |
| 发票 / 表单               | 5    | 自建          |
| 纯扫描件                  | 5    | 自建          |
| 公式密集文档              | 5    | 数学/物理论文 |
| 复杂表格文档              | 5    | 财报/统计报告 |
| 中日韩混排                | 5    | 自建          |
| 特殊布局（PPT/海报/手册） | 5    | 自建          |

**总计 60 份 PDF**

- [ ] 收集 / 生成评测数据集
- [ ] 为每份 PDF 准备 ground truth（人工标注或参考标准工具输出）
- [ ] Ground truth 格式定义（统一 JSON schema）

### 2. 评测指标

| 指标               | 说明                            | 目标     |
| ------------------ | ------------------------------- | -------- |
| **文本召回率**     | 正文文本提取完整度              | > 90%    |
| **文本精确率**     | 提取文本的正确性（无乱码/幻觉） | > 95%    |
| **阅读顺序准确率** | 段落/块的顺序是否正确           | > 85%    |
| **标题检测 F1**    | 标题区域检测的 F1 分数          | > 70%    |
| **表格检测 F1**    | 表格区域检测的 F1 分数          | > 80%    |
| **表格结构准确率** | 行列映射正确率                  | > 75%    |
| **公式检测召回率** | 公式区域检测完整度              | > 60%    |
| **RAG 检索命中率** | 基于扁平化索引的问答命中率      | > 80%    |
| **处理速度**       | 页/秒（CPU, release 模式）      | > 3 页/s |

- [ ] 实现每个指标的自动计算函数
- [ ] 生成对比报表（Markdown / HTML）

### 3. 自动化评测脚本

```bash
# 运行 knot-pdf 评测
./scripts/run_benchmark.sh --tool knot-pdf --dataset eval_v2/

# 运行 MinerU 评测（需要 Python 环境）
./scripts/run_benchmark.sh --tool mineru --dataset eval_v2/

# 生成对比报告
./scripts/compare_results.sh knot-pdf mineru
```

- [ ] MinerU 评测适配器（Python wrapper）
- [ ] 结果对比与报表生成

### 4. 对比报告内容

- [ ] 各指标得分对比表

- [ ] 差距归因分析（是模型差距还是规则差距？）
- [ ] 后续改进优先级建议

## 备注

本里程碑的核心目标不是"追平 MinerU 的每一项指标"（因为 MinerU 有 GPU 模型的优势），而是**明确差距在哪里、原因是什么、如何在保持轻量级定位的前提下缩小差距**。knot-pdf 的定位优势（毫秒级性能、零依赖部署、离线运行）应当被保留。
