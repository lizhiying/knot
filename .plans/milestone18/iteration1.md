Milestone: milestone18
Iteration: iteration1

Goal: 构建端到端的基于数据量的简单智能路由分发器，实现单表/小数据情况下的业务流。支持识别结构化数组数据，对其进行粗略 Token 估算，并在较小的时候交给直连 LLM，较大的时候交由后台单步 DuckDB SQL 引擎处理并取回答案。

Assumptions:
- 假定现阶段命中结构化结果只有一张表，不会命中多张（如果有，默认取第一张）。
- 假定 SQL 单词执行不会出错，一次取回即为结果。
- 检索出来其他非结构化文档噪音直接混入作为上下文处理，不进行过滤（后续迭代解决）。

Scope:
- 拆分检索结果链路识别结构化数据。
- 简单的 Token 估算（行数/长度等粗略算法）。
- 判断逻辑：估算结果与系统剩余上下文空缺进行对比。
- 实现一条自动分支逻辑，大表格转到原生的单条 DuckDB 处理，小型表转到内置分析 prompt 喂给直接 LLM。

Tasks:
- [x] 1. RAG 检索管线改造，添加对文本 Chunk 和结构化数组结果的鉴别与拆分逻辑。
- [x] 2. 结合后台上下文设计一种粗略极简的表格数据的 Token 容量估算法。
- [x] 3. RAG Router 引擎拦截并在预设 Token 阈值下走 Direct LLM，超高走 DuckDB。
- [x] 4. 跑通单次 SQL 询问测试，将 DuckDB 的执行结果返回并展现为文本。

Exit criteria:
- 用户抛出一张小 Excel 读取自然问题，系统顺利不加 DuckDB 进行 LLM 回答。
- 用户抛出一张大 Excel，系统正确启用 DuckDB，单步执行一条 SQL 并取得最终结果字符串。
