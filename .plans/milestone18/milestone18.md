# Milestone 18: 结构化数据（Excel/CSV）查询优化与交互重构

## 目标
使得 Knot 能够更智能、准确地响应针对 Excel/CSV 等结构化数据的自然语言查询。当数据量小时直接调用 LLM，数据量大时自动转用 DuckDB 执行 SQL 分析。在此过程中，妥善处理非结构化文档噪音、多表歧义，并通过多轮会话实现复杂查询的可视化展示。

## 成功标准 (Success Metric)
1. 系统可自适应根据数据量选择正确引擎 (LLM vs DuckDB)。
2. 检索并保留强相关的混合非结构化文档。
3. 命中多张表时成功暂停执行，给前端推列表让用户勾选，然后恢复执行。
4. SQL 执行失败或只完成一半意图时，LLM 能够与系统和用户循环交互并修正，直至展示出正确的数据表格。

## 迭代计划概览

*   **Iteration 1: End-to-end Routing & Basic SQL** 跑通最小闭环。实现数据类型解析与标记，构建简单的估算器，路由引擎；通过一个写死的选表或单表跳过机制跑通小数据的 LLM 直连及大数据单步 DuckDB SQL 执行。
*   **Iteration 2: 交互式表选择与降噪** 增强系统稳健性。引入前置 LLM 对混合文档的智能降噪判定；Router 实现“遇到多表变 Pending”机制，前台接入表格选择组件，选后回调完成闭环流程。
*   **Iteration 3: Agentic 多步执行与数据展示** 提升最终质量与体验。引入“Agentic Step-by-Step”的 SQL 执行代理流处理错误或追问操作；前台实现直接渲染呈现数据库查询结果原貌的 Grid 视图。
