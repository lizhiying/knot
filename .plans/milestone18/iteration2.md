Milestone: milestone18
Iteration: iteration2

Goal: 增强系统稳健性与防报错机制。在多表结果重叠时增加前端回调式待命选择；并对非结构化文件进行前置 LLM 降噪过滤，只提取高相关结果送入下文。

Assumptions:
- 假定上一步单表的路由选择机制运行正常。
- 假定混合文本降噪在 LLM 识别时不会消耗太长，且准度基本可用。

Scope:
- 混用文本数据的短判定 prompt 构建。
- 在命中多个 Sheet 或多次 CSV 同类结果时，拦截系统自动执行动作。
- 前后端对接的 `Pending/Select_Table` 数据结构回调交互流。

Tasks:
- [x] 1. 混合文本过滤机制：为文本 Chunk 构建相关性评估 Prompt，判定后过滤无用文档碎片再组合上下文发给主线流程。
- [x] 2. Router 对多结果拦截：一旦检出结构化结果 `count > 1` 将引擎设为 `Pending` 并给前端推送含有不同表名列表的信号。
- [x] 3. 前端接收信号并弹出表选择器。
- [x] 4. 回接信号：前端选中唯一表并触发回调，后端基于该单选项无缝衔接至由于未指定而被挂起的后续业务流。

Exit criteria:
- 查一个宽泛的行业词抛出一堆 PDF 杂质与一个关键 Excel 时，杂质被准确剔除而不是撑爆上下文。
- 当有 3 份 Excel 一并命中关键词引擎时，控制台/UI 会卡住等待用户明确选定“某一年表”而不是去执行死局或合并表数据。
