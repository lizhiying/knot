## 一、 产品需求文档 (PRD)

### 1. 产品定位

**Knot** 是一款基于 **Tauri** 构建的“Spotlight”风格本地 AI 助手。它通过深度索引本地文档（PDF/Markdown）、程序和系统 API，为用户提供毫秒级的全能搜索、本地文档问答、任务自动化执行功能。

### 2. 核心功能模块

#### A. 深度文档引擎 (PageIndex Inside)

* **智能索引：** 自动监控文件夹，利用 **OCRFlux-3B** 多模态模型解析 PDF（包含图片描述、表格结构还原）。
* **语义树构建：** 将长文档转化为层级清晰的 JSON 树，支持按章节、按摘要检索。
* **多模态搜索：** 用户可以通过“架构图”、“财务报表”等关键词搜到文档中的非文本元素。

#### B. 全能搜索 (Spotlight 增强版)

* **混合搜索：** 结合 BM25 关键词匹配与向量语义检索，确保既能搜到“确切文件名”，也能搜到“关于某个概念的段落”。
* **应用启动：** 快速定位并打开本地应用程序。

#### C. 本地 AI 问答 (RAG)

* **文档交互：** 基于检索到的上下文，调用 **Qwen-3** 进行总结、对比或细节查询。
* **隐私安全：** 所有推理流程 100% 离线，数据不离开用户硬盘。

#### D. Agent 自动化

* **自然语言指令：** “10点设置闹钟”、“添加买牛奶到待办事项”。
* **系统调用：** 自动通过操作系统 API 创建系统级提醒、待办或执行脚本。

### 3. 用户体验 (UX) 目标

* **唤起速度：** 快捷键唤起延迟 < 100ms。
* **搜索速度：** 10,000 个文档范围内检索延迟 < 50ms。
* **视觉风格：** 极简毛玻璃界面，支持主题自适应。

## 二、 技术架构文档

### 2. 技术栈选型

* **外壳框架：** **Tauri v2** (Rust + 现代前端框架如 Svelte/React)
* **核心逻辑：** **Rust** (高性能、内存安全)
* **向量数据库：** **LanceDB** (嵌入式、支持磁盘/内存混合模式)
* **感知模型 (VLM/OCR)：** **OCRFlux-3B** (通过 `llama.cpp` 运行 GGUF 量化模型)
* **认知模型 (LLM)：** **Qwen3 (0.6B)** (通过 `llama.cpp` 二进制侧载运行)
* **文本检索：** **Tantivy** (Rust 原生全文搜索引擎)

启动时预载 Embedding 引擎，LLM 采用 mmap 挂载以实现秒级热启动

### 3. 数据流设计

#### 索引阶段 (Ingestion Pipeline)

1. **文件监听：** Rust 后端 `notify` 模块监控磁盘变动。
2. **视觉解析：** 调用 OCRFlux-3B 识别 PDF 布局及内容（Text/Table/Figure）。
3. **结构化：** 参照 `page_index.py` 逻辑构建文档树。
4. **向量化：** 使用小型 Embedding 模型计算摘要向量，存入 LanceDB。

#### 检索阶段 (Retrieval Pipeline)

1. **意图路由：** 识别输入是“指令（Alarm/Todo）”还是“查询（Search）”。
2. **多级过滤：**
* L1：使用 `mmap` 加载的 LanceDB 进行 L1 文档层过滤。
* L2：对 Top-K 文档进行 PageIndex 树节点匹配。


3. **Prompt 构建：** 将检索到的树路径及正文片段封装为 Context。

### 4. 关键技术实现方案

#### 内存管理：`mmap` 与预温策略

* **策略：** LLM 模型文件通过 `llama.cpp` 的 `mmap` 方式挂载，避免强制占用物理内存。
* **触发器：** 监测到用户按键即触发模型“热点数据预读”，消除首次推理的 I/O 停顿。

#### 多模态还原：结构化视觉解析

* 利用 **OCRFlux-3B** 多模态模型直接生成高度结构化的 Markdown 内容，准确保留文档中的表格、标题层级和图片描述，减少后处理复杂度。
