# LLM 优化方案

| 方案            | 预期提升 | 复杂度 | 是否完成 | 说明                                                |
| :-------------- | :------- | :----- | :------- | :-------------------------------------------------- |
| 减少 n_predict  | 30-50%   | 低     | 是       | 改为 512-1024，回答更简短                           |
| 真流式输出      | 感知提升 | 中     | 是       | 前端边生成边显示，用户感知更快                      |
| Prompt Caching  | 首次加速 | 中     | 是       | 缓存上下文，消除 Prompt 处理耗时 (0.6s -> 0ms)      |
| 换更小模型      | 2-3x     | 高     | 是       | Qwen3-1.5B 代替 4B                                  |
| 模型预热        | 首次加速 | 低     |          | 启动时发一个空请求预热                              |
| Flash Attention | 10-20%   | 中     | 是       | llama.cpp 编译时启用 (已在启动参数通过 -fa on 启用) |

推荐：先实施 **方案2 (真流式输出)** + **方案3 (Prompt Caching)**，彻底解决“假死”感。

本地大模型推理速度更吃：
- 是否走 Metal / GPU 加速、实现是否优化
- 内存带宽
- GPU/加速器规模
- 模型大小、量化方式、上下文长度等
