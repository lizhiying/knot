UI: Knot RAG Spotlight

# Knot Spotlight UI 需求文档 (Design Spec)

## 1. 整体布局与视觉风格 (Visual & Layout)

### 1.1 容器设计

* **触发方式**：全局快捷键唤起（如 `Alt + Space`）。
* **位置**：屏幕水平居中，垂直偏上 (top 20%)，非全屏覆盖。
* **尺寸**：
* 初始状态（仅搜索框）：高度约 `60px - 80px`，宽度 `672px (max-w-2xl)`。
* 展开状态（有结果）：宽度扩展至 `896px (max-w-4xl)` 或 `1024px (max-w-5xl)`，高度根据内容自适应，最大高度 `80vh`。


* **背景风格**：
* **Glassmorphism (毛玻璃)**：高斯模糊背景，模拟 macOS 原生质感。
* **暗黑模式支持**：必须适配 Dark Mode。



### 1.2 布局结构 (Grid System)

一旦搜索开始，界面分为三个区域：

1. **Header (Search Input)**: 顶部常驻，用于输入。
2. **Main Content (Split View)**:
* **Left (60-65%)**: **"智" (AI Insight)** —— LLM 生成的答案流。
* **Right (35-40%)**: **"据" (Evidence)** —— 混合检索到的文档片段（Hybrid Search Results）。


3. **Footer**: 状态栏与快捷键提示。

## 2. 详细组件需求

### 2.1 搜索输入栏 (The Omni-Bar)

这是用户的第一接触点，需极简且流畅。

* **UI 元素**：
* 左侧：搜索图标（搜索中变为 Loading Spinner）。
* 中间：大号输入框，无边框，光标闪烁。
* 右侧：功能图标（清除按钮、设置齿轮）。


* **交互**：
* 输入时，下方内容区实时响应（或防抖响应）。
* 支持 `Tab` 键切换到结果列表进行键盘选择。



### 2.2 左侧：AI 生成区 (AI Insight)

这是用户视线的焦点。

* **状态展示**：
* **Thinking**: 显示“正在阅读 12 个文档...”、“正在思考...”的脉冲动画。
* **Streaming**: 文字逐字出现，光标跟随。


* **引用标注**：
* 生成的文本中包含引用角标 `[1]`。
* **交互**：鼠标悬停在 `[1]` 上时，右侧对应的文档卡片高亮；点击则滚动到该文档位置。


* **排版**：
* 支持 Markdown 渲染（粗体、列表、代码块）。
* 字体舒适，行高适中（Relaxed leading）。



### 2.3 右侧：检索结果列表 (Evidence List)

这是建立信任的关键，展示“匹配分”和“出处”。

* **卡片设计 (Result Card)**：
* **头部**：文件图标 + 文件名（截断）+ 面包屑（如 `... > 技术部 > 2023规划`）。
* **右上角**：匹配度徽章（Badge）。
* 高匹配 (>0.85): 绿色背景 `bg-green-100 text-green-700`。
* 中匹配: 黄色/灰色背景。


* **正文**：命中关键词的高亮片段（Snippet）。


* **排序**：按 Score 从高到低排列。
* **交互**：
* 点击卡片：调用系统默认应用打开该文件。
* Hover 效果：轻微上浮或背景变色。



### 2.4 底部状态栏 (Footer)

* **左侧**：显示当前索引状态（如“索引就绪：12,403 个文件”）。
* **右侧**：键盘快捷键提示（`⏎ 打开`，`↑↓ 选择`，`Esc 关闭`）。

## 4. 关键交互说明

1. **加载动画**：
* 在用户输入后、LLM 开始输出前，左侧区域展示 Skeleton Screen（骨架屏）或者简单的“思考中”脉冲动画。
* 右侧结果列表如果加载较慢，也应优先显示骨架。


2. **流式输出 (Streaming)**：
* 左侧文字必须是打字机效果（Typewriter effect）。
* 当 LLM 输出 `[1]` 时，系统自动在右侧列表中找到对应的卡片，并给予一次轻微的高亮闪烁，建立连接感。


3. **结果卡片**：
* **Snippet 截取**：不要显示文件的开头，而要显示包含关键词的那个句子及其前后文。
* **面包屑**：文件路径太长时，保留首尾，中间用 `...`，例如 `Documents/.../2024/Plan.docx`。


4. **键盘导航**：
* 当焦点在 Input 时，按下 `Down` 键，焦点移动到右侧列表的第一个结果。
* 在结果列表中，`Enter` 打开文件。
* 在结果列表中，`Left` 键回到 Input。
